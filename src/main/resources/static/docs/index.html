Content-Length: 4

인증 관련 Error Response

Example 1. 토큰이 없는 경우
HTTP/1.1 401 Unauthorized
Content-Type: application/json;charset=utf-8
Content-Length: 118

{
  "error_code" : 1000,
  "error_message" : "DEFAULT",
  "detail" : "You have to first either login or signup."
}
Example 2. 토큰이 유효하지 않는 경우

detail 은 달라질 수 있습니다.

HTTP/1.1 401 Unauthorized
Content-Type: application/json;charset=utf-8
Content-Length: 114

{
  "error_code" : 1002,
  "error_message" : "INVALID_JWT",
  "detail" : "Token does not match type Bearer."
}
Example 3. 토큰이 유효하지만, 해당 API나 리소스에 접근 권한 없는 경우

보통의 경우, 잘못된 API 경로나 Http Method 오류 가능성이 높습니다.

HTTP/1.1 403 Forbidden
Content-Type: application/json;charset=utf-8
Content-Length: 115

{
  "error_code" : 3001,
  "error_message" : "NO_ACCESS",
  "detail" : "You have no access to this resource."
}

Auth: 로그아웃

기존의 토큰이 만료되지 않습니다. Secure 쿠키를 없애는 용도로 사용하시면 됩니다.

Request
POST /logout HTTP/1.1
Host: docs.api.com
Content-Type: application/x-www-form-urlencoded
Response
HTTP/1.1 200 OK
Set-Cookie: refresh_token=; Path=/; Max-Age=0; Expires=Thu, 1 Jan 1970 00:00:00 GMT; HttpOnly; SameSite=None
